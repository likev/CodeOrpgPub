MRPG Glossary

ORPG Service Tasks

Service tasks are tasks that provide generic services for the ORPG system.   Service tasks are not terminated during normal mrpg shutdown procedures.  mrpg starts service tasks by executing the start_rssd script.  If additional service tasks are necessary, they should be added to the start_rssd script.  If a service task is listed in the task_table for mrpg monitoring, then the service will be terminated when the mrpg cleanup command is executed.  The remote system services daemon (rssd) is a service task responsible for providing event notification services and remote procedure call services for ORPG and MSCF machines.  The monitor network service task (mnet) is responsible for detecting network disconnections for ORPG and MSCF machines.    Refer to the start_rssd script for a complete list of ORPG service tasks.

ORPG Install Applications

Install applications are applications that complete ORPG installation duties that were not conveniently implemented in the ORPG install scripts.  mrpg executes install applications at every start up.  It is the install applications responsibility to insure that install duties are only executed when necessary.  Refer to the Install_cmds section of the task_table configuration file for a complete list of installation applications.  An install applications diagram also lists the installation tasks defined in the task_table configuration file. The task_table.4 man page describes the task_table configuration file. 

ORPG Startup Initialization Applications

Initialization applications are applications responsible for initializing ORPG data stores.  mrpg runs start up initialization tasks   sequentially when the "mrpg startup" command is invoked.  Refer to the Startup_init_cmds section of the task_table configuration file for a complete list of startup initialization applications.  A startup initialization applications diagram also lists the startup initialization applications defined in the task_table configuration file.  The task_table.4 man page describes the task_table configuration file.

ORPG Restart Initialization Applications

Initialization applications are applications responsible for initializing ORPG data stores.  mrpg runs re-start initialization tasks   sequentially when the "mrpg restart" command is invoked.  Refer to the Startup_init_cmds section of the task_table configuration file for a complete list of initialization applications.  A restart initialization applications diagram also lists the installation tasks defined in the task_table configuration file.   The task_table.4 man page describes the task_table configuration file.

ORPG Operational Applications

Operational applications are applications that run while mrpg is in operate state.   mrpg executes, monitors, restarts, and terminates operational applications whenever necessary.  Refer to the Operational_processes section of the task_table configuration file for a complete list of operational applications.  An operational applications diagram also lists the installation tasks defined in the task_table configuration file.  The task_table.4 man page describes the task_table configuration file.

ORPG Shutdown Commands

ORPG commands that are executed during the ORPG shutdown process.  mrpg runs these applications sequentially.  Refer to the Shutdown_cmds section of the task_table configuration file for a list of shutdown commands.  A  shutdown commands diagram also lists the installation tasks defined in the task_table configuration file.  The task_table.4 man page describes the task_table configuration file.

ORPG Data Stores

ORPG product data stores are defined in the product_tables configuration file.  Non-product data stores are defined in the data_tables configuration file.
